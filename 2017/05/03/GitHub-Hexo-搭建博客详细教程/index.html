GitHub + Hexo 搭建博客详细教程

发表于 2017-05-03

借鉴文章：最适合新手的 GitHub + Hexo 「大话」博客搭建教程 – smartbeng

  1. 前面说一两句，搭建博客可以先不考虑域名问题，本教程最终会绑定在Github上，只需要一个Github账号即可。
  2. 在下面教程需要输入的命令中，如 $ git config ，这里的「$」是命令前面的固定前缀，是不需要输入的，只需要输入 git config 即可。

安装Git

  1. Git 下载地址
  2. 安装 Git 后打开 git bash 界面（在文件夹中右键选择 Git Bash Here），以下所有命令操作全部在此窗口中进行。
  3. 设置 Git 的 user name 和 email
    1. 输入命令：$ git config --global user.name "username" ，这里的username替换为你自己的用户名
    2. 输入命令：$ git config --global user.email "XXX@163.com"，这里同理替换为自己的邮箱
    3. 通过输入命令 $ git config user.name、$ git config user.eamil，可以查看已经设置的用户名和邮箱。

Git SSH key 生成

  1. 首先检查你电脑上的 ssh key 密钥，输入命令 $ cd ~/. ssh
  2. 如果提示：No such file or directory 则说明电脑上没有密钥，继续进行操作；如果没有任何提示，则说明已经有 ssh 密钥，可以跳过下面第3步。
  3. 生成新的 SSH Key
    1. 输入命令 $ ssh-keygen -t rsa -C "haiyan.xu.vip@gmail.com"
      • 注意1：此处的邮箱地址，你要输入自己的邮箱地址
      • 注意2：此处的「-C」的是大写的「C」
    2. 输入第一步的命令，回车一次，然后系统会要求你输入密码
      • 这个密码会在你提交项目时使用，如果为空的话则提交项目时不需要输入密码。这个设置是为了防止别人往你的项目中提交内容。
      • 输入密码的时候没有「*」字样，直接输入即可。
    3. 没记错的话，还需要再按一次回车。如果没有异常信息出现，就成功生成了 ssh key。

添加 SSH Key 到 Github

  1. 生成 SSH Key 之后，找到下图所示的目录

    • 此文件夹内容为刚才生成的 SSH Key 密钥。如果看不到文件，需要设置显示隐藏的项目。
  2. 登陆 Github
  3. 点击右上角头像处的下拉列表 Settings -> SSH and GPG keys -> 右上角 New SSH key
  4. 在第一步的目录中找到名为 id_rsa.pub 的文件，右键选择使用记事本打开，全选里面的内容复制之后，粘贴到刚刚打开的网页中（key文本框中），点击 add key 就完成了。

使用Github 建立博客

  1. 登陆Github，在Github首页，点击页面右下角的「New Repository」
  2. 填写项目信息：
    • Repository name：username.github.io
      注意：username为你的用户名，即页面右上角点击你的头像，显示出来的用户名
    • Description (optional)：填写项目描述，可以不填
    • 点击「Create Repository」完成创建。
    • Ps：Github Pages的Repository name是特定的，比如我的Github账号的用户名是 CanChengZheng，那么我Github Pages Repository（Github仓库）名字就是CanChengZheng.github.io
      每个账号只能有一个仓库来存放个人主页，而且仓库的名字必须是username.github.io，这是特殊的命名约定。完成之后你可以通过 username.github.io 来访问你的个人主页。
    • 如果仓库名字创建错了，可以查看下面的链接进行删除，重新再进行创建。

安装Hexo

Hexo的作者是Tommy Chen，根据官方介绍，Hexo是一个简单、快速、强大的博客发布工具，支持MarkDown格式。

  1. 在「我的电脑」中选择一地方新建「Hexo」文件夹，比如在E盘新建一个文件夹「Hexo」
    • 这里的文件夹名没有硬性要求，这里我们取名「blogtest」，方便后面讲解
    • 该文件夹用于存放Hexo工具，以及以后写的博文也会先在这个文件夹中进行编辑）
  2. 进入该文件夹，右键 -> Git Bash Here 打开 Git，输入命令 $ npm install -g hexo，根据网络情况，可能需要等到一段时间，安装成功，Git 界面如下图所示。
  3. 接下来输入命令 $ hexo init 回车运行，Hexo会在目标文件夹建立网站所需要的所有文件。同样根据网络情况，可能需要等待一段时间，才能完成。
    初始化成功后，Git 界面如下图所示。
  4. 现在已经搭建起了一个本地博客，输入一下命令进行验证
    1. $ hexo g 生成博客
    2. $ hexo s 启动服务本地预览（预览完通过键盘按下 Ctrl + C 停止本地预览）
    3. 打开浏览器，输入localhost:4000，一个初始博客就显示出来了。

新建文章

  1. 先介绍几个命令
    1. hexo new "postName" 新建文章，postName是新建的文章的名字，内容就在这里面编写
    2. hexo new page "pageName" 新建页面
  2. 生成的文章会存在于 “你的博客目录\source_posts” 目录下
  3. 文章默认的格式是.md，使用的是 markdown 语法，推荐一个不错的 Markdown编辑器 - MarkdownPad 2
  4. 如果暂时没学过Markdown，可以先随便写点什么上去，ctrl + s 保存
  5. 现在对新生成的文章进行预览
    1. 确认Git界面目录切换到博客目录下（即一开始取名的 blogtest 文件夹中，对应地，你切换自己的文件夹下）
    2. 使用以下命令，生成网页，启动本地预览服务，打开浏览器查看效果（http://localhost:4000/）

  2. 输入下列命令，clone我们自己的Github仓库到本地

    • 以后需要部署本地的博客到Github，可以直接双击这个文件，在弹出的窗口中输入Github的账号和密码，就可以推送上去了。

补充：

  1. 以后发布博客只需要三步：
    1. hexo new “postname” 新建并编写博文，
    2. hexo g 生成博客
    3. 双击 deploy.sh 脚本文件，输入账号密码即可完成发布。
  2. 如果更换了主题，记得先 hexo clean，再 hexo g 生成博客
  3. _config.yml 文件中的 timezone 配置不能错
    • timezone: zh-CN #能配置成 +08:00 类似的
      上面也有不对的地方，如果设置成 +08:00 后momentjs解析会报异常，需要将timezone配置成时区名称timezone: Asia/Shanghai
  4. 关于域名的绑定，博客现在是绑定在Github的域名上的，可以通过你的username.github.io访问你的博客，如果将来你买了自己的域名，可以将这个域名绑定到Github上，就可以通过你购买的域名访问你的博客了，这是后话了，具体可以自行Google，就不再详述了。

Hexo常用的几个命令

  1. hexo generate (hexo g)
    生成静态文件，会在当前目录下生成一个新的名为 public 的文件夹
  2. hexo server (hexo s)
    启动本地web服务，用于博客的预览
  3. hexo deploy (hexo d)
    部署博客到远程服务器上，比如github，heroku等平台
  4. hexo new “postName”
    新建文章，postName是新建的文章的名字，内容就在这里面编写
  5. hexo new page “pageName”
    新建页面
