Git 命令

运行 Git 前的配置

  1. 配置用户信息
    • $ git config –global user.name “ZCC”
    • $ git config –global user.email zcc550169544@163.com
    • 如果用了 –global 选项，那么更改的配置文件就是位于你用户主目录下的那个，以后你所有的项目都会默认使用这里配置的用户信息。如果要在某个特定的项目中使用其他名字或者电邮，只要去掉 –global 选项重新配置即可，新的设定保存在当前项目的.git/config文件里。
  2. 查看配置信息
    • $ git config –list
    • 直接查阅某个环境变量的设定，只要把特定的名字跟在后面即可，
      eg. $ git config user.name
  3. 获取帮助
    • 想了解 Git 的各种工具怎么使用，可以通过输入命令阅读它们的使用帮助，格式：$ git help
      eg. $ git help config

Git 基础

取得项目的 Git 仓库

  1. 从当前目录初始化
    • $ git init
  2. 从现有的仓库克隆
    • $ git clone [url]
      eg. 克隆Ruby语言的 Git 代码仓库：$ git clone git://github.com/schacon/grit.git
    • 克隆的时候，自己定义要新建的项目目录名称，可以在上面的命令最后指定：
      $ git clone git://github.com/schacon/grit.git mygrit
    • Git 支持许多数据传输协议，详情见Pro git P14 ​

记录每次更新到仓库

  1. 检查当前文件状态
    • 要确定哪些文件当前处于什么状态：$ git status
  2. 跟踪新文件
    • $ git add README
  3. .gitignore格式规范
    • Pro git P18
  4. 查看具体修改了什么地方
    • $ git diff
    • 详情见Pro git P19
  5. 提交更新
    • $ git commit
      这种方式会启动文本编辑器来输入本次提交的说明。
    • $ git commit -m
      使用 -m 参数后跟提交说明的方式，在一行命令中提交更新。
  6. 移除文件
    • $ git rm
    • 详情见Pro git P19
  7. 移动文件
    • $ git mv

查看提交历史

详情见Pro git P24

  1. 回顾提交历史
    • $ git log
    • –p 选项展开显示每次提交的内容差异
    • -2 仅显示最近的两次更新
      $ git log –p -2
    • –stat 仅显示简要的增改行数统计
    • –pretty 可以指定使用完全不同于默认格式的方式展示提交历史，详情见Pro git P26

撤销操作

详情见Pro git P29

  1. 修改最后一次提交
    • $ git commit –amend
  2. 取消已经暂存的文件
    • $ git reset HEAD …
  3. 取消对文件的修改
    • $ git chechout – …

远程仓库的使用

  1. 查看当前的远程库
    • $ git remote，列出每个远程库的简短名字
    • -v 选项，显示对应的克隆地址
  2. 添加远程仓库
    • $ git remote add [shortname] [url]
  3. 从远程仓库抓取数据
    • $ git fetch [remote-name]
    • fetch 命令只是将远端的数据拉到本地仓库，并不自动合并到当前的工作分支，当你准备好了，才能手工合并。
  4. 自动抓取数据
    • $ git pull
    • 详情见Pro git P33
  5. 推送数据到远程仓库
    • $ git push [remote-name] [branch-name]
  6. 查看远程仓库信息
    • $ git remote show [remote-name]
  7. 远程仓库的删除和重命名
    • 重命名：$ git remote rename pb paul
    • 删除：$ git remote rm paul

打标签

  1. 显示已有的标签
    • $ git tag
  2. 创建含附注的标签
    • $ git tag -a v1.4 -m ‘my version 1.4’
    • -a 指定标签名字
    • -m 指定对应的标签说明
    • $ git show v1.4 查看相应标签的版本信息
  3. 创建轻量级标签
    • $ git tag v1.4
